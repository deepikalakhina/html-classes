In submit input type, we dont use label for display, the text that needs to be displayed is added inside the value. Here, the user can click the submit button, but the data will not be stored. since this input type is Synchronous loading. When we clicked on the submit button, the user was redirected to a fresh page which was loaded after discarding the current page hence its called Synchronous loading.

Here, we used "action" property / attribute inside the form tag so that when clicked on submit button, an action could be performed.

Checkbox allows multiple selection within the defined set of otpions.

Label is only for the front display and Value is for us to internal see what option the user has selected.

For radio button type, browser will allow the user to choose both the options by default. Adding the tag "name" and defining its value for ex - name="gender", the user will now be able to choose only 1 option. Here gender is a variable. inside 1 variable only 1 value can be set. in this case, we have male and female - so when we give name to any radio and give same name then it has common variable. since its a variable it allows only 1 value to be set

In input type button, when we add only value, when clicked it doesnt do any action.

Every input elements has events for example , checkbox - check, uncheck are the events.

To detect the change on the input elements, we use events.

For button, common event used is onclick. we will also give value inside onclick to tell the browser what to do when clicked. general output element is alert. In alert, we use bracket and use single quotes and enter the message that needs to be displayed. Alert is a blocking element. untill you click on the ok button, you cannot do anything on that page.
